## 1. 项目概述

### 1.1 项目背景
本项目是一个虚拟炒股平台，旨在为用户提供一个模拟股票交易的环境，用户可以在平台上使用虚拟资金进行股票买卖练习，了解股票市场的运作和投资理财知识。

### 1.2 项目目标
- 为用户提供真实的股票交易体验
- 支持多群组、多用户管理
- 提供完整的数据分析功能
- 实现后台管理功能

### 1.3 用户角色
| 角色 | 说明 | 权限 |
|------|------|------|
| 普通用户 | 注册后加入群组进行交易 | 买卖股票、查看统计 |
| 群组管理员 | 群组内用户 | 查看群组成员交易 |
| 系统管理员 | 后台管理者 | 全部管理权限 |

### 3.2 核心业务流程

#### 3.2.1 用户注册流程
```
[用户] -> [输入用户名密码] -> [输入邀请码] -> [验证]
    -> [创建用户] -> [加入群组] -> [初始化资金] -> [完成]
```

#### 3.2.2 股票交易流程
```
[用户] -> [选择群组] -> [选择股票] -> [输入数量]
    -> [验证：是否今天已交易]
    -> [验证：余额是否充足]
    -> [获取昨日收盘价]
    -> [创建交易记录]
    -> [更新持仓]
    -> [更新资金]
    -> [标记今日已交易]
    -> [完成]
```

##### 3.2.3 群组消息与股票信息展示
```
[用户A买入贵州茅台] -> [系统自动生成群组消息]
    -> [消息显示: 用户A、操作类型(买入)]
    -> [消息显示: 股票代码、股票名称、市场类型]
    -> [消息显示: 单价、总价、创建时间]
    -> [其他群组成员可点赞、回复]
```

每个交易操作（买入/卖出/分红/配股）自动生成群组消息，展示完整的股票信息。

#### 3.2.4 群组成员详情查看
```
[用户进入群组页] -> [排名列表加载各成员资产]
    -> [选择某个成员] -> [查看成员详情]
    -> [资金摘要: 可用资金、总资产、收益、收益率]
    -> [持仓列表: 代码、名称、市场、数量、均价、现价、市值、浮盈亏]
    -> [交易记录: 代码、名称、类型、数量、价格、金额、日期]
```

### 3.2.5 收益计算
```
可用现金 = 资金余额
持仓市值 = Σ(持股数量 × 当前价格 × 汇率换算)
总资产 = 可用现金 + 持仓市值
收益 = 总资产 - 初始资金
收益率 = (收益 / 初始资金) × 100%
```

### 3.2.4 货币换算
系统统一使用 **人民币(CNY)** 作为记账单位。

| 市场 | 原始货币 | 换算汇率 | 说明 |
|------|----------|----------|------|
| A股 | CNY | × 1 | 直接使用 |
| 港股 | HKD | × 0.9 | 约等于 CNY |
| 美股 | USD | × 7 | 汇率 |

### 3.2.5 股票行情缓存
为避免频繁调用外部API，系统采用缓存机制：

1. **首次获取**：从新浪/雅虎API获取实时行情
2. **缓存存储**：保存到 `stock_prices_cache` 表（当天有效）
3. **后续获取**：优先从缓存读取，隔天自动更新
4. **历史记录**：同时保存到 `stock_prices` 表

```javascript
// 缓存逻辑伪代码
if (cache.trade_date === today) {
  return cache  // 直接返回缓存
} else {
  quote = fetchFromAPI()
  saveToCache(quote)
  saveToHistory(quote)
  return quote
```

### 3.2.6 持仓与资金模型
持仓和资金独立于群组，统一计算：

- **持仓表**：按用户+股票维度，不区分群组
- **资金表**：按用户维度，不区分群组
- **群组**：仅用于统计展示，不影响交易计算

## 10. 后台管理系统

### 10.1 系统概述

后台管理系统是为平台管理员提供的独立管理界面，与用户交易界面分离。

**设计原则**:
- 独立登录入口确保安全
- 需管理员授权才能访问
- 支持完整的用户管理、股市配置、统计功能

### 10.2 功能模块

| 模块 | 功能 |
|------|------|
| 用户管理 | 查看/控制用户、登录历史、交易记录 |
| 群组管理 | 查看/创建/编辑群组、成员管理 |
| 股票管理 | 股票池管理、手动刷新、缺失检查 |
| 市场配置 | 刷新时间、交易时间设置 |
| 统计中心 | 群组收益排名、用户活跃度统计 |
| 邀请码 | 生成/管理邀请码 |
| 佣金设置 | 各市场佣金比例配置 |

### 10.3 技术架构

**认证方式**:
- 独立后台登录 `/api/v1/admin/login`
- 使用 JWT Token (7天有效期)
- 存储在 adminToken

**前端路由**:
- 登录页: `/admin-login`
- 管理页: `/admin`
- 路由守卫检查 adminToken

### 10.4 多语言国际化

系统支持简体中文(zh-CN)、繁体中文(zh-TW)、英文(en)三种语言。

**实现方式**:
- 前端使用 vue-i18n v9 的 Composition API 模式 (`legacy: false`)
- 语言文件存放在 `frontend/src/locales/` 目录
- 所有页面文字通过 `$t('key')` 调用，script 中使用 `t('key')`
- 语言选择器放在首页和管理页面顶部导航栏
- 语言偏好通过 `localStorage` 持久化

**触发变更**:
- 切换语言时通过 `watch(() => i18n.global.locale.value, ...)` 监听变化
- 自动更新 Element Plus 组件库的 locale
- 切换即时生效，无需刷新页面
