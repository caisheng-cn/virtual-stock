# 测试方案文档

## 1. 概述

本文档描述虚拟炒股平台的测试框架、测试策略和实施计划。采用 BDD（行为驱动开发）风格进行测试编写。

## 2. 测试框架选型

| 层级 | 框架 | 说明 |
|------|------|------|
| 后端单元测试 | Jest | 成熟稳定的 JavaScript 测试框架 |
| 后端集成测试 | Jest + supertest | Mock 数据库，测试 API 路由 |
| 后端端到端测试 | Jest + supertest | 真实数据库，完整 API 流测试 |
| 前端组件测试 | Vitest + @vue/test-utils | 与 Vite 项目契合 |

## 6. CI/CD 集成

测试在每次 push 和 pull request 时自动运行。

```yaml

    runs-on: ubuntu-latest
    steps:
      - name: Setup Node.js

    steps:
      - name: Run frontend tests
        run: npm test
```

## 7. 覆盖率目标

| 项目 | 初期目标 | 长期目标 |

### 7.1 覆盖重点

**后端优先级**:
1. 交易计算逻辑（买入/卖出金额）
2. 行情数据解析（A股/港股/美股）
3. 用户验证和权限
4. 资金余额计算

**前端优先级**:
1. 表单验证逻辑
2. 计算属性（金额计算）
3. 组件交互逻辑
4. 错误处理

## 8. BDD 风格指南

### 8.1 结构

```
describe('功能模块 - 模块名称', () => {
  describe('Given 场景描述', () => {
    describe('When 操作描述', () => {
      it('Then 预期结果', () => {
        expect(actual).toBe(expected)
      })
    })
  })
})
```

### 8.2 常用关键词

- **Given** (假设): 测试的前置条件
- **When** (当): 用户执行的操作
- **Then** (那么): 预期的结果
- **And** (并且): 补充条件

### 8.3 示例

```javascript
describe('Given 用户已登录', () => {
  describe('And 用户有足够余额', () => {
    describe('When 买入100股贵州茅台', () => {
      it('Then 应扣除相应金额', () => {
        expect(newBalance).toBe(originalBalance - amount)
      })

      it('And 应创建持仓记录', () => {
        expect(position.shares).toBe(100)
      })
    })
  })
})
```

## 10. 注意事项

1. **E2E 测试依赖**: E2E 测试需要后端服务运行在 localhost:3001
2. **数据库状态**: 测试可能修改数据库，建议使用单独的测试数据库
3. **外部 API**: 行情数据依赖新浪/雅虎 API，网络异常可能导致测试失败
4. **测试顺序**: E2E 测试假设某些数据已存在（如 testuser）

## 11. 故障排除

### 11.1 常见问题

| 问题 | 解决方案 |
|------|----------|
| 数据库连接失败 | 检查 MySQL 是否运行，配置是否正确 |
| API 超时 | 增加 testTimeout 或检查网络 |
| Mock 失效 | 检查 jest.mock 导入路径 |
| Element Plus 组件报错 | 使用 global.stubs stub 第三方组件 |

## 14. 附录

### A. Jest 常用命令

```bash
jest --watch              # 监视模式
jest --coverage           # 生成覆盖率报告
jest --testNamePattern="" # 按名称过滤测试
jest --testPathPattern=""  # 按路径过滤测试
jest --bail               # 遇到失败停止
```

### B. Vitest 常用命令

```bash
vitest watch             # 监视模式
vitest run --coverage     # 单次运行 + 覆盖率
vitest --testNamePattern  # 按名称过滤
```

### C. supertest 响应断言

```javascript
// 状态码
```
