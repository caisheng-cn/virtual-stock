## 3. 核心业务逻辑说明

### 3.1 开户流程
1. 用户注册 -> 创建用户记录
2. 用户加入群组 -> 在user_groups添加记录
3. 初始化资金 -> 在user_balance插入记录，cash = group.init_cash

### 3.2 交易流程
1. 检查用户状态是否正常
2. 检查今天是否已交易（last_trade_date != today）
3. 获取股票昨日收市价
4. 检查余额是否充足（买入时）
5. 检查持仓是否充足（卖出时）
6. 创建交易记录
7. 更新持仓表
8. 更新用户资金表
9. 更新用户最后交易日期

### 3.3 收益计算
- 总资产 = 现金 + 持仓市值
- 收益 = 总资产 - 初始资金
- 收益率 = (总资产 - 初始资金) / 初始资金 * 100%

### 3.4 货币换算规则

系统统一使用 **人民币(CNY)** 作为记账单位。

| 市场 | 原始货币 | 换算汇率 | 说明 |
|------|----------|----------|------|
| A股 | CNY | × 1 | 直接使用，无需换算 |
| 港股 | HKD | × 0.9 | 港币按 0.9 折算为人民币 |
| 美股 | USD | × 7 | 美元按 7 折算为人民币 |

**示例**：
- 贵州茅台价格 1800 CNY → 直接使用 1800 CNY
- 腾讯控股价格 400 HKD → 换算为 400 × 0.9 = 360 CNY
- Apple 价格 180 USD → 换算为 180 × 7 = 1260 CNY

**交易金额计算**：
- 买入金额 = 换算后单价 × 股数
- 卖出金额 = 换算后单价 × 股数
- 持仓成本 = 换算后单价 × 股数（记录在 positions 表）

**市值计算**：
- 持仓市值 = Σ(股数 × 换算后单价)

### 3.5 佣金规则

系统对每笔交易收取佣金，买卖均收取。佣金配置保存在 `commission_configs` 表中，支持按市场单独设置。

**佣金配置表：commission_configs**

| 字段名 | 数据类型 | 说明 |
|--------|----------|------|
| id | INT | 主键 |
| market_type | TINYINT | 市场类型：1A股 2港股 3美股 |
| trade_type | TINYINT | 交易类型：1买入 2卖出 |
| commission_rate | DECIMAL(10,6) | 佣金比例 |
| updated_at | DATETIME | 更新时间 |

**默认佣金比例**：

| 市场 | 买入/卖出 | 佣金比例 |
|------|-----------|----------|
| A股 | 买入 | 万分之5 (0.0005) |
| A股 | 卖出 | 万分之5 (0.0005) |
| 港股 | 买入 | 千分之1 (0.001) |
| 港股 | 卖出 | 千分之1 (0.001) |
| 美股 | 买入 | 千分之5 (0.005) |
| 美股 | 卖出 | 千分之5 (0.005) |

**佣金计算公式**：
- 买入：股票金额 = 单价 × 股数，佣金 = 股票金额 × 买入佣金比例，实际扣款 = 股票金额 + 佣金
- 卖出：股票金额 = 单价 × 股数，佣金 = 股票金额 × 卖出佣金比例，实际到账 = 股票金额 - 佣金

**佣金存储**：
- 每笔交易记录中保存 `commission`（佣金金额）和 `commission_rate`（本次交易佣金比例）

**佣金历史表：commission_history**
> 佣金比例变更历史记录，每次修改都会保存完整历史

| 字段名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | INT | 是 | AUTO_INCREMENT | ID |
| market_type | TINYINT | 是 | - | 市场类型：1A股 2港股 3美股 |
| trade_type | TINYINT | 是 | - | 交易类型：1买入 2卖出 |
| old_rate | DECIMAL(10,6) | 是 | - | 原佣金比例 |
| new_rate | DECIMAL(10,6) | 是 | - | 新佣金比例 |
| changed_by | INT | 是 | - | 操作用户ID |
| changed_at | DATETIME | 是 | CURRENT_TIMESTAMP | 修改时间 |
| remark | VARCHAR(200) | 否 | - | 修改备注 |

**索引**：
- `market_type, trade_type, changed_at`
- `changed_at`

**说明**：
- 修改佣金比例时自动记录原比例、新比例、操作用户、时间、备注
- 支持追溯任意历史变更

## 4. 命名规范

### 4.1 表命名
- 所有表名使用小写字母
- 多词用下划线分隔
- 复数形式命名（如users, groups, positions）

### 4.2 字段命名
- 小驼峰命名法（username, userId）
- 时间字段用 created_at, updated_at
- 金额用 decimal(15,2)
- 比率用 decimal(10,4)

### 4.3 索引命名
- 主键：pk_表名
- 唯一索引：uk_表名_字段名
- 普通索引：idx_表名_字段名
