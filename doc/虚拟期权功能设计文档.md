# 虚拟期权交易功能 - 设计文档

> 版本：v1.0  
> 日期：2026-05-13

---

## 1. 概述

### 1.1 项目目标

在现有虚拟股票交易平台基础上，增加虚拟期权交易功能。用户可以对白名单内的标的股票进行期权模拟交易，体验买入开仓、卖出平仓、提前行权、到期结算等操作。

### 1.2 功能范围

| 项目 | 方案 |
|------|------|
| 市场风格 | 美股期权风格（美式，到期前可提前行权） |
| 标的范围 | 管理员指定的白名单股票 |
| 到期周期 | 按日滚动（每日都有不同到期日的合约） |
| 行权价 | 基于标的价格自动生成（±5档，每档2.5%） |
| 定价模型 | 简化固定价差（内在价值 + 时间价值） |
| 操作类型 | 买入开仓(BTO) → 卖出平仓(STC)，不做卖方 |
| 保证金 | 预留机制（当前仅买方交易，暂无需保证金） |
| 交割方式 | 现金结算（到期按价差自动结算盈亏） |

### 1.3 约定术语

| 术语 | 说明 |
|------|------|
| 权利金 (Premium) | 买入期权时支付的价格 |
| 行权价 (Strike) | 期权约定的买卖价格 |
| 到期日 (Expiration) | 期权最后有效日 |
| Call | 看涨期权 |
| Put | 看跌期权 |
| 实值 (ITM) | 有内在价值的期权 |
| 平值 (ATM) | 行权价接近当前价的期权 |
| 虚值 (OTM) | 无内在价值的期权 |

### 2.2 表关系图

```
option_whitelist ──── 标的股票 ──── stock_pools
      │
      │ 白名单中的股票才能生成合约
      ▼
option_contracts ──── 合约定义 ──── option_prices (每日行情)
      │
      ├── 用户持有 ──→ option_positions
      │
      └── 交易记录 ──→ option_transactions
      
option_positions ──→ user_balance (共用现金账户)
option_transactions ──→ user_balance (资金变动)
```

### 2.3 现有表改动

**user_balance**：无需改表。期权市值计入 `total_assets` 的计算逻辑中（在服务层处理）。

**transactions**：保持不变。期权交易记录独立存储于 `option_transactions`，与股票 `transactions` 表分开查询，前端可合并展示。

**group_ranking_cache**：无需改表。期权市值和盈亏纳入总资产和总利润计算即可。

### 3.3 交易业务流程

#### 买入开仓 (Buy to Open)

```
1. 验证用户状态、交易权限、交易时间
2. 查询合约：校验合约状态为"交易中"、未到期
3. 获取最新权利金报价（option_prices）
4. 计算总权利金 = 权利金 × 张数 × 合约乘数(100)
5. 计算手续费 = 总权利金 × 手续费率
6. 总扣款 = 总权利金 + 手续费
7. 校验 user_balance.cash >= 总扣款
8. 扣减 cash（DB事务内）
9. 创建/更新 option_positions（均价 = (原有成本 + 新成本) / 总张数）
10. 写入 option_transactions（trade_type=1）
11. 发群组消息（message_type=5）
```

#### 卖出平仓 (Sell to Close)

```
1. 验证用户状态、交易权限
2. 查询 option_positions：校验持仓足够
3. 获取最新权利金报价
4. 计算收入 = 权利金 × 张数 × 合约乘数
5. 计算手续费 = 收入 × 手续费率
6. 净收入 = 收入 - 手续费
7. 计算盈亏 = 净收入 - 卖出部分的成本
8. 增加 cash（DB事务内）
9. 扣除持仓数量，归零时删除 position
10. 写入 option_transactions（trade_type=2, 记录profit）
11. 发群组消息（message_type=6）
```

#### 提前行权 (Exercise)

```
1. 查询 option_positions：校验持仓足够
2. 获取标的最新价
3. 计算结算金额（看涨 = (股价 - 行权价) × 张数 × 乘数；看跌 = (行权价 - 股价) × 张数 × 乘数）
4. 校验结算金额 > 0（只有实值期权可以行权）
5. 增加 cash（加结算金额）
6. 扣除 option_positions
7. 写入 option_transactions（trade_type=3, 记录 settlement_amount 和 profit）
8. 发群组消息（message_type=7）
```

#### 到期自动结算 (Expiration Settlement) — 定时任务

```
每天收盘后执行：
1. 查询所有到期且未结算的合约
2. 对每张合约，查询其所有持仓用户
3. 对每个持仓：
   a. 获取标的收盘价
   b. 计算内在价值（= 结算金额）
   c. 若虚值（内在价值=0）则作废，不结算
   d. 若实值则 cash += 结算金额
   e. 更新 position 状态为"已到期"
   f. 写入 option_transactions（trade_type=4, 记录profit）
   g. 发群组消息（message_type=8）
4. 更新合约状态为"已到期"
```

## 4. 定价模型

### 4.1 简化公式

```
权利金 = 内在价值 + 时间价值

内在价值 (Call) = max(标的价 - 行权价, 0)
内在价值 (Put)  = max(行权价 - 标的价, 0)

时间价值 = 行权价 × 年化波动率 × √(剩余天数/365) × 系数

系数说明：
  - 平值期权(ATM): 系数 = 1.0（时间价值最大）
  - 实值期权(ITM): 系数 = 0.3 ~ 0.7（越实值越小）
  - 虚值期权(OTM): 系数 = 0.5 ~ 0.9（越虚值越小）
```

### 4.2 参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 年化波动率 | 30% | 可针对不同标的调整 |
| 行权价档位 | ±5档 | 当前价上下各5个行权价 |
| 行权价间距 | 股价×2.5% | 每档之间的价格差 |
| 合约乘数 | 100 | 每张期权对应100股 |
| 期权手续费率 | 0.5‰ | 同A股手续费率 |

### 4.3 合约生成逻辑

#### 入口

合约生成有两个触发入口：

| 入口 | 触发方式 | 说明 |
|------|----------|------|
| **管理后台手动生成** | 管理员点击「生成合约」按钮 | POST `/api/v1/admin/options/contracts/generate` |
| **每日定时任务** | 每个交易日 09:35 自动执行 | 遍历白名单所有启用的标的 |

两个入口最终都调用相同的 `ensureContracts()` + `refreshPrices()` 函数。

#### 执行链路

```
用户/定时器
    │
    ▼
ensureContracts(stockCode, marketType, stockName, underlyingPrice)
    │
    ├─ 1. 查询 option_contracts 获取该标的所有未到期合约的到期日列表
    │     （WHERE stock_code=? AND expiration_date >= today AND status=1）
    │
    ├─ 2. 计算缺失的到期日
    │     从今天起未来 5 周，每周一个到期日（+7天, +14天, ..., +35天）
    │     跳过已有合约的到期日（不重复生成）
    │
    ├─ 3. 生成 11 个行权价
    │     基准价 = 标的价（四舍五入到档位步长）
    │     从 -5档 到 +5档，每档间距 = 标的价 × 2.5%
    │     例如 AAPL @ $294.80 → $257.95, $265.32, ..., $331.65
    │
    ├─ 4. 组合生成合约 → 写入 option_contracts 表
    │     遍历: 每个(到期日 × Call/Put × 行权价)
    │     使用 findOrCreate 防止重复：contract_code 是唯一键
    │     如果已存在则跳过，不存在则创建新合约
    │
    │     contract_code 编码规则:
    │       标的代码 + 到期日(YYYYMMDD) + C/P + 行权价×1000
    │       例: AAPL20260520C00294800
    │
    │     一次生成数量: 5到期日 × 2类型 × 11行权价 = 110 条记录
    │
    ▼
refreshPrices(stockCode, marketType)
    │
    ├─ 1. 从 stock_prices_cache 获取标的当日收盘价
    │
    ├─ 2. 查询 option_contracts 获取该标的所有活跃合约
    │
    ├─ 3. 遍历每张合约，调用 calcPremium() 计算：
    │      权利金 = 内在价值 + 时间价值
    │      通过 upsert 写入 option_prices 表
    │      （contract_id + trade_date 唯一键，每天每合约只有一条）
    │
    └─ option_prices 字段：
         contract_id       - 合约ID（外键）
         trade_date        - 日期
         premium           - 权利金
         intrinsic_value   - 内在价值
         time_value        - 时间价值
         underlying_price  - 标的当日收盘价
```

#### 涉及的数据表

| 表 | 操作 | 说明 |
|----|------|------|
| `option_whitelist` | 读取 | 获取启用的标的列表 |
| `stock_prices_cache` | 读取 | 获取标的当前价（定价基准） |
| `option_contracts` | 写入 | `findOrCreate` 合约定义，已存在不重复 |
| `option_prices` | 写入 | `upsert` 每日权利金报价 |

## 6. 资金账户交互

### 6.1 共用 user_balance 机制

期权交易直接使用现有 `user_balance` 表的 `cash` 字段：

| 操作 | 对 cash 的影响 | 备注 |
|------|---------------|------|
| 买入开仓 | `cash -= 总权利金 + 手续费` | 支付权利金 |
| 卖出平仓 | `cash += 净收入` | 收回权利金+盈亏 |
| 提前行权 | `cash += 结算金额` | 现金交割 |
| 到期结算 | `cash += 结算金额` | 自动执行 |

### 6.2 总资产计算

现有 `total_assets = cash + stock_market_value`

调整为：`total_assets = cash + stock_market_value + option_market_value`

其中：
- `stock_market_value` = 各股票持仓的市值之和（已有逻辑）
- `option_market_value` = 各期权持仓的市值之和 = Σ(权利金最新价 × 张数 × 合约乘数 × 汇率)

### 6.3 对现有接口的影响

| 接口 | 改动 |
|------|------|
| `GET /api/v1/balance` | `totalAssets` 加入期权市值；`profit` 加入期权浮动盈亏 |
| `GET /api/v1/statistics/profit` | 加入期权已实现盈亏和浮动盈亏 |
| `GET /api/v1/statistics/positions` | 加入期权持仓统计 |
| `GET /api/v1/groups/:id/members/:userId/details` | 返回数据增加 `optionPositions` 字段 |

## 8. 定时任务

在 `src/services/` 中新增 `optionSync.js`，通过 `node-cron` 或 `setInterval` 调度：

| 任务 | 触发时机 | 说明 |
|------|----------|------|
| 生成新合约 | 每个交易日 09:35 | 检查次日到下次周度的合约，补齐 |
| 刷新期权报价 | 每个交易日每隔5分钟 | 重新计算所有活跃合约的权利金 |
| 到期结算 | 每个交易日 16:30 | 处理当日到期的合约持仓 |
| 标记到期合约 | 每个交易日 16:35 | 更新 option_contracts.status=2 |
